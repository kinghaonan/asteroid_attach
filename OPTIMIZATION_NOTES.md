# 优化更新说明

## 2025-02-13 优化内容

### 1. 打印输出优化

#### 问题
- 样本计算时每个batch都打印进度，刷屏严重
- 模型训练时每次保存都打印"模型保存完成"，重复打印
- 字体警告影响观感

#### 解决方案

**样本计算进度**
- 引力计算：每500个点打印一次进度（原来是每100个）
- 梯度计算：每200个点打印一次进度（原来是每100个或每个batch）

**模型保存**
- 训练过程中静默保存最佳模型（verbose=False）
- 训练结束后再打印一次最终模型保存信息
- 新增 save_model(verbose=False) 参数控制

**字体设置**
- 移除 SimHei 字体，改用系统默认 sans-serif
- 避免中文字体缺失字形警告

### 2. 训练输出格式优化

**旧格式**：
```
Epoch [  5/20] | 总损失: 1.234567e-03 | 引力损失: 5.678901e-04 | 梯度损失: 6.543210e-04 | 验证总损失: 1.111111e-03
```

**新格式**：
```
Epoch [  5/20]  Train: 1.2346e-03  Val: 1.1111e-03  (g:5.68e-04, grad:6.54e-04)
```

更简洁，一行显示关键信息

### 3. 样本复用优化

- 新增 `reuse_existing: true` 配置
- 计算过的样本自动保存到 `data/samples/gravity_samples.npz`
- 下次运行时自动加载，跳过重复计算
- 同时保存引力和梯度数据

## 使用效果

### 第一次运行
```
📊 生成新的引力样本（同时计算引力和引力梯度）...
   样本数量: 500
   采样范围: 1.1 - 5.0 倍半径
   计算引力加速度...
   引力计算: 500/500 (100.0%)
   计算引力梯度...
   梯度计算: 200/500 (40.0%)
   梯度计算: 400/500 (80.0%)
   梯度计算: 500/500 (100.0%)
```

### 第二次运行（复用）
```
📂 发现已存在的样本文件，正在加载...
✅ 样本加载成功: 500 个点
   包含引力: (500, 3) 和梯度: (500, 9)
```

**节省计算时间约 80-90%！**

## 配置文件示例

```yaml
phase1:
  sampling:
    num_samples: 500
    reuse_existing: true  # 设为true启用复用
    sample_file: "data/samples/gravity_samples.npz"
  
  training:
    epochs: 20
    batch_size: 64
    print_freq: 5  # 每5个epoch打印一次
```

## 下一步建议

1. **减少样本数量**：快速测试时设为 100-200
2. **增加样本数量**：正式验证时设为 5000-10000
3. **启用复用**：正式运行前先用小样本测试流程，然后增加样本数量复用计算结果
